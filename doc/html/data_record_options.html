Data Record Options and Status

This page describes the options and status for the data recorder.

Aurora Remote SDK provides a set of data recorder to help user record the data from an Aurora device.

Data Recorder Types

Aurora Remote SDK provides the following data recorder types:

  • SLAMTEC_AURORA_DATARECORDER_TYPE_RAW_DATASET
    • Record the raw data from the Aurora device.
  • SLAMTEC_AURORA_DATARECORDER_TYPE_COLMAP_DATASET
    • Record the sparse reconstruction data and the related images in colmap format. User can further use the colmap to reconstruct a dense 3D model or perform 3d gaussian splatting.

Options and Status Format

The options are stored in a map, the key is the option name, the value is the option value.

The option name and value are case-sensitive.

The option value can be a null-terminated string, a 32-bit integer, a 64-bit integer, a double-precision floating-point number, or a boolean value.

As C language do not support boolean type, the boolean value is represented as 0 or 1.

The status is similar to the options, but the value is the current status of the data recorder.

Recording Options

Common Options

  • image_quality
    • To specify the image stream type to record.
    • **"raw"** (default)
      • Record raw image stream. Best quality but high bandwidth.
    • **"preview"**
      • Record preview image stream. (compressed)

Colmap Recorder Options

  • stereo_recording
    • To specify whether to record stereo image stream.
    • false (default)
      • Record mono image stream.
    • true
      • Record stereo image stream.
  • undistort
    • To specify whether to undistort the image.
    • false (default)
      • Do not undistort the image.
    • true
      • Undistort the image.
  • undistort_force_focal_center
    • To specify whether to force the focal center to be the center of the image.
    • false (default)
      • Do not force the focal center to be the center of the image.
    • true
      • Force the focal center to be the center of the image.
  • keep_unused_map_points
    • To specify whether to keep unused map points.
    • false (default)
      • Remove unused map points.
    • true
      • Keep unused map points.
  • multi_mapper
    • Controls whether the related sparse reconstruction data is stored in a sparse/n/ folder.
    • false (default)
      • Store the related sparse reconstruction data in a sparse/ folder.
    • true
      • Store the related sparse reconstruction data in a sparse/n/ folder.
  • file_format
    • **"binary"** (default)
      • Store the data in binary format.
    • **"text"**
      • Store the data in text format.
    • **"all"**
      • Store the data in both binary and text format.

Available Status
