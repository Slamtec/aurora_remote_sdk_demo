SDK Setup and Usage

This document covers the API description of SLAMTEC Aurora Remote C/C++ SDK.

About the SDK

The SDK contains the following components:

  • header files for the SDK API
  • pre-compiled dynamic libraries for the SDK targets various operating systems and architectures

The headers provide interfaces and data types definitions written in C++ 14 and C.

In order to use the SDK, you need to include the header files and link against the pre-compiled dynamic libraries.

Supported Platforms

The SDK supports the following platforms:

  • Linux : x86_64, arm64(aarch64)
  • Windows (x86_64)

More platforms will be supported in the future. If you need other platforms, please contact SLAMTEC.

Getting Started

Use SDK Demo Project

There is a SDK Demo project for you to get started quickly. We are highly recommended you to use the demo project to understand how to use the SDK.

Prerequisites

  • C++ 14 compiler

    The C++ sdk requires a C++ compiler that supports C++ 14 standard. For example, GCC 7.5 or above, or MSVC 2019 or above.

  • C compiler

    The C sdk requires a C compiler that supports C99 standard.

  • Windows specific

    There is no specific requirement for what MSVC compiler version to use the SDK, any version that supports C++ 14 and C99 standard will be fine.

    But as the pre-compiled dynamic libraries are built with MSVC 17 (VS2022) using /MD option. You should download related VC runtime libraries first to make the SDK work, or simply install the Visual Studio 2022. The VC runtime libraries can be downloaded from here.

Setup the include path

Please add the path that contains the file aurora_pubsdk_inc.h to your compiler's include path.

For example, if you are using GCC, you can add the following command to your compiler command line:

-I/path/to/aurora/public_remote_sdk/inc

Setup the library path and link the libraries

Add the path that contains the pre-compiled dynamic libraries to your compiler's library path.

Please note that the library path is different for different platforms.

For example, if you are using GCC on a x86_64 Linux machine, you can add the following command to your compiler command line:

-L/path/to/aurora/public_remote_sdk/lib/linux_x64

Link the pre-compiled dynamic libraries to your project.

The pre-compiled dynamic libraries are named as libslamtec_aurora_remote_sdk.so for Linux, and AuroraPubSDK_C.dll and slamtec_aurora_remote_sdk.dll and slamtec_aurora_remote_sdk.lib for Windows.

For example, if you are using GCC on a x86_64 Linux machine, you can add the following command to your compiler command line:

-lslamtec_aurora_remote_sdk

You should add the static library slamtec_aurora_remote_sdk.lib to your project if you are using MSVC on Windows.

Deployment

The pre-compiled dynamic libraries only depends on the C++ standard library, so you don't need to deploy any other dependencies.

But make sure to deploy the related VC runtime libraries if you are using MSVC on Windows.

Basic Usage

Here is a simple example to show how to use the SDK to connect to a SLAMTEC Aurora device and get the device pose.

Please refer to the SDK Demo project for more examples.

// include the SDK header file, both C and C++
#include "aurora_pubsdk_inc.h"
// other dependencies
#include <iostream>
// the namespace of the SDK
using namespace rp::standalone::aurora;
int main(int argc, char** argv) {
// the sdk session should be created before using any other APIs, except for GetSDKInfo
RemoteSDK * sdk = RemoteSDK::CreateSession();
// connect to the device, the default device ip address is "192.168.11.1"
// Alternatively, you can use the sdk->getDiscoveredServers() to automatically discover the device ip address
bool ret = sdk->connect({"192.168.11.1"});
if (!ret) {
std::cerr << "Failed to connect to the device" << std::endl;
return -1;
}
// get the device pose
slamtec_aurora_sdk_pose_t pose;
// Note: it is recommended to get Pose in SE3 format (getCurrentPoseSE3), which is more accurate
// The following function get pose in euler angle format, which may have gimbal lock issue
if (!sdk->dataProvider.getCurrentPose(pose)) {
std::cerr << "Failed to get the device pose" << std::endl;
return -1;
}
std::cout << "Current pose: " << pose.translation.x << ", " << pose.translation.y << ", " << pose.translation.z
<< " Euler: " << pose.rpy.roll << ", " << pose.rpy.pitch << ", " << pose.rpy.yaw << std::endl;
// the sdk session should be destroyed before the program exits
RemoteSDK::DestroySession(sdk);
// alternatively, you can use the release() method to release the sdk session
// sdk->release();
return 0;
}
