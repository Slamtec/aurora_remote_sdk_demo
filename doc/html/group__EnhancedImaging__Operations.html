const void *  raw_segment_data,
slamtec_aurora_sdk_image_desc_t *  desc_out,
const slamtec_aurora_sdk_enhanced_imaging_frame_buffer_t *  aligned_segment_data 
)

Calculate the aligned segmentation map.

Caller can use this function to calculate the aligned segmentation map.

The function will calculate the aligned segmentation map from the raw segmentation map that matches the coordinate of the depth map.

The caller should provide the buffer of the raw segmentation map and the buffer of the aligned segmentation map.

The function will return SLAMTEC_AURORA_SDK_ERRORCODE_OK if the aligned segmentation map is calculated successfully.

Parameters
handle- the session handle
desc_in- the description of the raw segmentation map
raw_segment_data- the buffer of the raw segmentation map
desc_out- the description of the aligned segmentation map
aligned_segment_data- the buffer of the aligned segmentation map
Returns
the error code

slamtec_aurora_sdk_errorcode_t AURORA_SDK_API slamtec_aurora_sdk_dataprovider_depthcam_peek_related_rectified_image ( slamtec_aurora_sdk_session_handle_t  handle,
uint64_t  timestamp,
slamtec_aurora_sdk_enhanced_imaging_frame_desc_t *  frame_desc_out,
const slamtec_aurora_sdk_enhanced_imaging_frame_buffer_t *  frame_buffer 
)

Peek the related rectified image of the current depth camera frame.

Caller should provide a timestamp to get the related rectified image.

If the timestamp is too old, the function will return SLAMTEC_AURORA_SDK_ERRORCODE_NOT_READY.

The caller must provide a buffer to store the rectified image.

The buffer size must be at least as large as the size of the rectified image.

The function will return SLAMTEC_AURORA_SDK_ERRORCODE_OK if a new rectified image is available.

Parameters
handle- the session handle
timestamp- the timestamp
frame_desc_out- the frame description
frame_buffer- the frame buffer
Returns
the error code

int  enable,

)

Set the post filtering of the depth camera.

Post filtering is used to refine the depth estimation from the depth camera module. It is enabled by default.

Caller can use this function to enable or disable the post filtering.

Parameters
handle- the session handle
enable- the enable flag, if set to non-zero, the post filtering will be enabled, otherwise it will be disabled
flags- the flags to control the post filtering, currently it is not used, set to 0

◆ slamtec_aurora_sdk_dataprovider_depthcam_wait_next_frame()

slamtec_aurora_sdk_errorcode_t AURORA_SDK_API slamtec_aurora_sdk_dataprovider_depthcam_wait_next_frame ( slamtec_aurora_sdk_session_handle_t  handle,
uint64_t  timeout_ms 
)

Wait for the next depth camera frame.

Caller can use this function to wait for the next depth camera frame.

The function will block until a new frame is available or the timeout occurs.

The function will return SLAMTEC_AURORA_SDK_ERRORCODE_NOT_READY if the timeout occurs.

The function will return SLAMTEC_AURORA_SDK_ERRORCODE_OK if a new frame is available.

Parameters
handle- the session handle
timeout_ms- the timeout in milliseconds, if set to -1, the function will block until a new frame is available
Returns
the error code
